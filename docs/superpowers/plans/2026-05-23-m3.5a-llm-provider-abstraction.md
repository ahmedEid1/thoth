# M3.5a — LLM Provider Abstraction Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Atlas's Anthropic-direct LLM layer with a Vercel-AI-SDK-based provider abstraction so that Google Gemini (free) is the default, with one-flag swap to Anthropic, OpenAI, or Groq. All 70+ existing tests continue to pass with mocks.

**Architecture:** Keep `runLLM<T>(args)` as the single public LLM call surface. Internally it becomes a dispatcher that resolves a **provider** (gemini/anthropic/openai/groq) and a **tier** (smart/fast) into a concrete `ai`-SDK model instance, then calls `generateObject` with the existing Zod schemas. Prompt builders move from Anthropic-typed `TextBlockParam[]` / `MessageParam[]` to provider-neutral `string` system + `ModelMessage[]`. Langfuse tracing migrates from the direct `langfuse` JS SDK to the OpenTelemetry exporter (`langfuse-vercel`), with explicit flushes preserved for Trigger.dev runtime.

**Tech Stack:** Vercel AI SDK (`ai` v5+), `@ai-sdk/google`, `@ai-sdk/anthropic`, `@ai-sdk/openai`, `@ai-sdk/groq`, `langfuse-vercel`, `@vercel/otel`, `@opentelemetry/api`. Removes: `@anthropic-ai/sdk`.

**Spec:** `docs/superpowers/specs/2026-05-23-m3.5-free-tier-pivot-design.md`

**Builds on:** v0.3.0-m3 (commit `1788f62`)

**Ship target:** `v0.3.5-m3.5a` tagged on GitHub, ≥70 tests passing, zero real LLM calls.

```ts
export type Tier = "smart" | "fast";
export type ProviderName = "gemini" | "anthropic" | "openai" | "groq";

const MODELS: Record<ProviderName, Record<Tier, string>> = {
  gemini: {
    smart: "gemini-2.5-pro",
    fast: "gemini-2.5-flash",
  },
  anthropic: {
    smart: "claude-opus-4-7",
    fast: "claude-sonnet-4-6",
  },
  openai: {
    smart: "gpt-4o",
    fast: "gpt-4o-mini",
  },
  groq: {
    smart: "llama-3.3-70b-versatile",
    fast: "llama-3.1-8b-instant",
  },
};

export function resolveTier(tier: Tier, provider: ProviderName): string {
  const tierMap = MODELS[provider];
  if (!tierMap) throw new Error(`Unknown LLM provider: ${provider}`);
  const model = tierMap[tier];
  if (!model) throw new Error(`Unknown tier "${tier}" for provider "${provider}"`);
  return model;
}
```

```ts
import type { LanguageModel } from "ai";
import { geminiModel } from "./gemini";
import { anthropicModel } from "./anthropic";
import { openaiModel } from "./openai";
import { groqModel } from "./groq";
import type { ProviderName } from "@/lib/llm/tiers";

export type ModelFactory = (modelId: string) => LanguageModel;

const ADAPTERS: Record<ProviderName, ModelFactory> = {
  gemini: geminiModel,
  anthropic: anthropicModel,
  openai: openaiModel,
  groq: groqModel,
};

export function resolveProvider(name: ProviderName): ModelFactory {
  const factory = ADAPTERS[name];
  if (!factory) throw new Error(`Unknown LLM provider: ${name}`);
  return factory;
}
```

```ts
  tier: Tier;
  /** Max output tokens. */
  maxTokens: number;
  /** System prompt (single string — provider-neutral). */
  system: string;
  /** Conversation messages. */
  messages: ModelMessage[];
  /** Zod schema for structured output. */
  schema: z.ZodType<T>;
  /** Trace metadata (runId, projectId, userId, etc.). */
  metadata?: Record<string, unknown>;
};

export type RunLLMResult<T> = {
  output: T;
  /** Trace URL is now constructed via Langfuse OTel exporter — populated by instrumentation. */
  traceUrl: string;
  usage: {
    inputTokens: number;
    outputTokens: number;
    totalTokens: number;
    /** Cache read tokens, when the provider reports them. 0 otherwise. */
    cacheReadInputTokens: number;
  };
};

/**
 * The single LLM call surface for the whole codebase.
 *
 * Responsibilities:
 *   - Resolve {provider, tier} → concrete ai-SDK LanguageModel
 *   - Call generateObject with the caller's Zod schema (SDK enforces it)
 *   - Attach experimental_telemetry so the Langfuse OTel exporter captures the span
 *   - Surface a uniform RunLLMResult regardless of provider
 *
 * NOT responsible for:
 *   - Flushing Langfuse: the OTel exporter is flushed by either Next.js' after()
 *     (in app routes) or by Trigger.dev's task-end hook (in trigger/*).
 */
export async function runLLM<T>(args: RunLLMArgs<T>): Promise<RunLLMResult<T>> {
  const providerName = env.LLM_PROVIDER as ProviderName;
  const factory = resolveProvider(providerName);
  const model = factory(resolveTier(args.tier, providerName));

  const meta = args.metadata ?? {};
  const runId = typeof meta.runId === "string" ? meta.runId : undefined;

  const { object, usage } = await generateObject({
    model,
    schema: args.schema,
    system: args.system,
    messages: args.messages,
    maxOutputTokens: args.maxTokens,
    experimental_telemetry: {
      isEnabled: true,
      functionId: args.name,
      metadata: {
        ...meta,
        sessionId: runId,
        tags: [args.tier, providerName],
      },
    },
  });

  return {
    output: object as T,
    // Trace URL is recorded on the OTel span; left empty here. Consumers that
    // need a URL can construct one from env.LANGFUSE_HOST + the captured span ID
    // (see lib/llm/telemetry.ts in M3.5b). Empty string is intentional placeholder.
    traceUrl: "",
    usage: {
      inputTokens: usage.inputTokens ?? 0,
      outputTokens: usage.outputTokens ?? 0,
      totalTokens: usage.totalTokens ?? 0,
      cacheReadInputTokens: 0,
    },
  };
}
```
